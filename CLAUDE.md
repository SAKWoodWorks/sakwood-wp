## Project Structure

This is a monorepo for **Sakwood**, a WordPress + Next.js e-commerce site for premium wood products in Thailand.

```
sakwood/
└── sakwood-wp/
    ├── frontend/           # Next.js 16 frontend (React 19)
    ├── wordpress-plugin/   # WordPress/WooCommerce integration plugin
    ├── nginx/              # Production Nginx configs
    ├── docker-compose.yml  # Dev environment
    └── docker-compose.prod.yml # Production environment
```

## Quick Start

**For complete documentation, see:** `sakwood-wp/CLAUDE.md`

**Development:**
```bash
cd sakwood-wp

# Start WordPress, MySQL, phpMyAdmin
docker-compose up -d

# Start frontend (separate terminal)
cd frontend
npm run dev
```

**Production Deployment:**
```bash
cd sakwood-wp
./deploy.sh          # Mac/Linux
.\deploy.ps1         # Windows PowerShell
```

## Architecture Overview

**Tech Stack:**
- **Frontend:** Next.js 16 (React 19) with App Router
- **Backend:** WordPress 6.4+ + WooCommerce
- **API:** WPGraphQL + Custom REST endpoints (`/wp-json/sakwood/v1/*`)
- **Database:** MySQL 5.7 (dev), MySQL 8.0 (prod)
- **Deployment:** Docker containers

**Key Architectural Patterns:**

1. **Dual-Service Pattern:** Every feature has two service files:
   - `lib/services/featureService.ts` - Server-side (SSR)
   - `lib/services/featureServiceClient.ts` - Client-side (interactive)

2. **Language-Prefixed Routing:** All API calls must use `/${language}/wp-json/...` prefix
   - Prevents query parameter loss from middleware redirects

3. **Environment Split:**
   - Server-side: `WORDPRESS_API_URL=http://sak_wp:80/wp-json` (includes `/wp-json` for menuService.ts)
   - Client-side: `NEXT_PUBLIC_WORDPRESS_API_URL=https://wp.sakww.com/wp-json/sakwood/v1` (full path)

## Critical Production Issues

### Docker Networking (Containers Can't Communicate)

**Symptom:** Frontend container cannot reach WordPress container.

**Diagnosis:**
```bash
# Check if containers share a network
docker inspect sak_frontend | grep -A 10 Networks
docker inspect sak_wp | grep -A 10 Networks
```

**Solution:** Ensure `docker-compose.prod.yml` has:
```yaml
networks:
  sakwood_network:
    driver: bridge

services:
  wordpress:
    networks:
      - sakwood_network
  frontend:
    networks:
      - sakwood_network
```

**Restart services:**
```bash
cd /var/www/sakwood
docker-compose -f docker-compose.prod.yml down
docker-compose -f docker-compose.prod.yml up -d
```

## Environment Files

**Development** (`sakwood-wp/frontend/.env.local`):
```env
NEXT_PUBLIC_WORDPRESS_GRAPHQL_URL=http://localhost:8006/graphql
NEXT_PUBLIC_WORDPRESS_API_URL=http://localhost:8006/wp-json/sakwood/v1
```

**Production** (`sakwood-wp/frontend/.env.production`):
```env
NEXT_PUBLIC_WORDPRESS_GRAPHQL_URL=https://wp.sakww.com/graphql
NEXT_PUBLIC_WORDPRESS_API_URL=https://wp.sakww.com/wp-json/sakwood/v1
```

**Note:** Server-side env vars (`WORDPRESS_API_URL`) are set in `docker-compose.prod.yml`, not in `.env` files.

## Development Workflow

**Frontend changes:**
```bash
cd frontend
npm run dev   # Hot reloads at http://localhost:3000
```

**Note:** `frontend/next-env.d.ts` is auto-generated by Next.js and should not be manually edited. It's frequently modified during builds and updates.

**WordPress plugin changes:**
```bash
# Hot-reload into running container
docker cp wordpress-plugin/sakwood-integration/file.php \
  sak_wp:/var/www/html/wp-content/plugins/sakwood-integration/
```

**Database operations:**
```bash
# phpMyAdmin: http://localhost:8888 (root/sakWW099)
# Or via CLI:
docker exec -it sak_db mysql -uroot -psakWW099
```

## Common Issues

**Frontend can't connect to WordPress:**
```bash
docker-compose ps                    # Verify WordPress running
curl http://localhost:8006/graphql   # Test GraphQL
```

**Plugin changes not appearing:**
```bash
docker restart sak_wp
```

**Build errors:**
```bash
cd frontend && rm -rf .next && npm install
```

## Key Files Reference

**Frontend Configuration:**
- `frontend/next.config.ts` - Rewrite rules, image optimization

- `nginx/frontend.conf` - Frontend Nginx config
- `nginx/wordpress.conf` - WordPress Nginx config
